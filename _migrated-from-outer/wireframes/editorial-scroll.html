SCROLL ARCHITECTURE — IMPLEMENTATION NOTES

Emotional arc: Arrival → Curiosity → Understanding → Surprise → Connection → Empathy → Trust → Clarity → Action → Belonging → Anticipation → Investment

Content sources: All editorial content from Strapi (dynamic zones). All transactional data from Supabase (funding amounts, bids, activity feed, gating). Video from Mux. Images from Cloudinary/R2.

Scroll behaviors: Subtle fade-in on text blocks (IntersectionObserver, not aggressive). Inline videos autoplay muted when scrolled into view. Full-width images have slight parallax (subtle — 5-10% movement, not jarring). Progress bar in nav updates as you scroll AND as funding changes. No scroll-jacking. No horizontal scrolling. Natural vertical scroll throughout.

The only persistent UI: The nav bar with progress bar + amount + timer + CTA. Everything else is editorial content that scrolls naturally. No sticky sidebars, no floating elements, no pop-ups.

Dark theme: The page should be dark (near-black background, light text) to let the photography and video dominate. The content is the color. The UI is invisible.

Typography: Distinctive serif for headlines and pull quotes. Clean sans for body and UI. TBD by design — the wireframe deliberately has no font opinions.

Mobile: Everything stacks. Person features go image-above-text. Tiers go single column. Activity feed simplifies. The core experience (video → story → people → ask → social proof) is the same scroll on every device.
